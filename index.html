What does a 2.2M-parameter LSTM
actually predict?

A next-word model trained from scratch on a 16GB laptop — Embedding(50) → LSTM(128) → Dense(12,000, softmax). No cherry-picked outputs. Every bar below is the model's real softmax probability.

Heads up: this model was trained on ~10,000 short sentences for 30 epochs — a small dataset for a 12,000-word vocabulary. Its raw next-word confidence is often only a few times better than random chance, and generated sequences will frequently be incoherent rather than grammatical. That's the model being honest about its own uncertainty, not a bug in this demo.

There are some points you should consider while generating the Word:

1.The backend is deployed on the free tier of Render so it can take 30-60sec to generate the first text.

2.Changing the temperature value can change the result like low temperature (e.g 0.15,0.25) can generate the sequence grametically close but high temperature introuce the creativity so it can generate the creative words.

3. As I mentioned earlier that this LSTM model was trained on a very small dataset so do not comment "Kya faltu cheej bnai h" or something.Bcz i trained this model and it took me 4hrs to just pass 30epochs so just generate the text and chillll....

4. This WebPage will only show you how an LSTM output is generated by SOFTMAX and how Temperature can affect it.
